# Thread 写作

把想法或素材变成一串能抓人、读得下去、有落点的 X/Twitter thread。

核心信念:**好的 thread 是用"钩子 + 干货 + 案例 + 观点 + CTA"五种积木搭出来的。** 每条有自己的作用,少了哪块读者就会掉队。

写 thread 的底层心法,简化成三步:

1. **第一条给极强的钩子**——抓住注意力,否则后面白写
2. **中段给有价值的信息或故事**——让读者觉得看完有得着
3. **核心亮点自然呈现**——不硬推,降低营销感

后面所有细节(五种积木、组合模板、风格规则)都是为这三步服务。写的时候可以复杂,但心里要一直记着这三步。

---

## 什么时候用这个 skill

- "帮我写个 thread / 写一串"
- "这个想法/文章/笔记,改成 thread"
- "我要发一串介绍 XX"
- 用户给了素材(笔记、bullet points、产品介绍、文章草稿)要做成 thread

**不适用**:
- 单条短帖(这个 skill 专做多条串联)
- 长文/博客/公众号文章

## 工作流程

### 第一步:判断输入类型

| 输入 | 应对 |
|------|------|
| **粗糙想法**("我想写个关于 XX 的 thread") | 先追问 1-2 个关键点,挖出他真正想说的那一句,再生成 |
| **已有素材**(笔记、文档、产品介绍) | 读完素材,提炼 1 个核心观点 + 3-5 个可用点,再往积木里塞 |

### 第二步:先定"一句话核心"

动笔前先用一句话说清楚:**这个 thread 在讲什么、为什么别人要看完。**

这一句通常不出现在 thread 里,但它是整个 thread 的脊柱。说不出这一句,thread 一定散。

### 第三步:选组合模式

根据一句话核心的类型,选一个模板起手。下面两组模板可以叠用——上面一组按"积木语言"搭,下面一组按"场景语言"搭,挑适合的那组。

**按内容类型(积木视角)**:

| 内容类型 | 组合 |
|---------|------|
| 教程/方法 | 钩子(痛点) → 案例(效果展示) → 干货 ×N → CTA(链接) |
| 产品/作品发布 | 钩子(悬念/数据) → 案例(效果) → 干货(功能) → CTA(链接 + 互动) |
| 经验分享 | 钩子(痛点) → 干货(方法 ×N) → 观点(感悟) → CTA(预告) |
| 观点/思考 | 钩子(反常识/争议) → 干货(论证) → 观点(延伸) → CTA(互动) |

**按 thread 场景(对标视角)**:

| 场景类型 | 组合 |
|---------|------|
| 产品发布型 | 公告 → 功能拆解 → 演示 → CTA |
| 用例展示型 | 痛点 → 多场景演示 → 真实案例 |
| 技术发布型 | 技术细节 → 性能数据 → 对比图表 → 文档链接 |
| 教程型 | 步骤拆解 → 每步配视觉示例 → 展示最终效果 |

模板是起点不是枷锁,根据素材灵活调。

### 第四步:按积木顺序填内容

每条都要能回答:**它是哪种积木?它在这里的作用是什么?**

五种积木的细分、例子、使用场景见 `references/building-blocks.md`。

### 第五步:整体检查

过"交付前自查"清单。特别是第一条——第一条读不下去,后面再好也没人看。

## 五种积木速查

详见 `references/building-blocks.md`。

| 积木 | 作用 | 位置 |
|------|------|------|
| **钩子** | 抓注意力 | 必须第一条 |
| **干货** | 核心内容 | 中间主体 |
| **案例** | 增强说服力 | 紧跟钩子或干货 |
| **观点** | 建立连接 | 灵活穿插 |
| **CTA** | 引导行动 | 结尾 |

子类:

- **钩子**:痛点共鸣 / 反常识 / 悬念 / 数据冲击 / 争议性开场
- **干货**:方法论 / 步骤拆解 / 对比分析 / 核心功能罗列
- **案例**:效果展示 / 前后对比 / 数据佐证
- **观点**:个人感悟 / 思考延伸 / 争议性观点
- **CTA**:链接引导 / 互动邀请 / 预告钩子 / 转发邀请

## 写作风格规则

这些规则违反了会让 thread 立刻变味。

### 结构原则

- **第一条必须概括核心信息**——读者只看第一条也要能知道这 thread 讲什么
- **后续逐步展开具体要点**——从钩子到细节,有递进逻辑
- **每一条都要提供独立的价值**——不要出现只是铺垫、只是过渡的条,那种条直接删掉
- **从头到尾有清晰的递进逻辑**——前后顺序换一下如果还读得通,说明逻辑没立起来
- **用 `1/ 2/ 3/` 编号**——条与条之间的连贯靠编号建立

### 钩子(首条)原则

- **以抓眼球的陈述开头**——数据、反差、结论前置
- **有明确的价值主张**——读者要立刻知道看完能得到什么
- **避免泛泛而谈**——"最近研究了一下 XX""分享一些关于 XX 的想法"这种开头=自杀
- **制造好奇心或紧迫感**——让读者觉得不看下去亏了

### 降低营销感

最高级的产品/工具类 thread 读起来不像广告。做法:

- **让核心亮点在演示和案例里自然浮现**,不要在第一条就喊"最强""最快""最好用"
- **先讲痛点和场景,再讲产品**——读者先认同问题,才会接受解法
- **用真实数据和细节替代形容词**——"省 40 分钟"比"极大提升效率"可信十倍
- **适当暴露缺点或局限**——100% 正面=不可信
- **观点和个人感受可以有,但不要用力过猛**——"挺好用"比"改变了我的人生"更可信

### 简洁直接,砍到最短

- 一条 thread 单元 1-3 句,**能一句说完就别两句**
- 一条只讲一个点,有第二个点就拆
- 删掉"其实""大概""可能""或许"这种缓冲词
- 删掉"首先、其次、最后"这种连接——用独立的条和编号代替
- 书面语铺垫词全删:"综上所述""值得注意的是""让我们""众所周知"

### 具体 > 抽象

- "提升效率" → "原来 2 小时,现在 10 分钟"
- "有很多人" → "评论区三十多条"
- 数字、时间、场景越具体越好

### 留白 > 塞满

- 一条里别把话说完,留点给下一条
- 结尾不要总结("以上就是...")——这是公众号习惯,thread 里很蠢
- 偶尔一条就一句话,反而有节奏

### Emoji 和符号(用但克制)

符号是 thread 的节奏工具,用得准能提质感,乱用降档次。规则:

**👇**:用在"接下来看这个"——下一条是配图、链接、效果展示时。一个 thread 最多出现 1-2 次。

**编号 `1/ 2/ 3/`**:列方法、列步骤、列功能时首选。比 emoji bullet(🔥✨💡)专业得多。

**`·`(中点)**:短列表、并列项。适合 2-4 项的轻量罗列。

**`→`(箭头)**:流程、因果、对比。"A → B → C" 比 "先 A,再 B,最后 C" 清爽。

**emoji 整体克制**:一条里 1-2 个,不要堆砌。装饰性(🔥✨💡🚀)和情绪性(😂🤣😭)都可以用,但一条里 emoji 多到抢戏就过了——当表情符号多到让读者先看 emoji 再看字时,减量。

### 节奏感

- 条和条之间空行分开
- 长条短条交错——3 条都是长段落,读者会累
- 偶尔插一条极短的(一句话),像呼吸

## 交付前自查

写完每个 thread 都过一遍:

1. **第一条能单独成立吗?** 读者只看第一条,愿不愿意点第二条?
2. **第一条泛了没?** "分享一些想法" "最近研究了" 这种开头立刻重写
3. **有没有一条可以删掉?** 删掉不影响阅读就删——纯过渡、纯铺垫的条不该存在
4. **每条都有独立价值吗?** 每条都能单拎出来做一个小点
5. **最长的一条能不能拆两条?** 通常能
6. **有没有书面语赘词?** "此外""值得一提""综合来看""让我们"——全改掉
7. **有没有具体的数字/场景/例子?** 全抽象的 thread 没人看
8. **营销感重不重?** 通篇"最强""改变人生"这种形容词就重写,换成具体数据和场景
9. **结尾是 CTA 还是总结?** 是总结就改 CTA 或删掉
10. **emoji 有没有抢戏?** 读者是不是会先看 emoji 再看字?是就减

## 输出格式

默认这样输出:

```
【一句话核心】
(提炼的主线,让用户看清 thread 在讲什么)

【组合】
钩子(xx 类) → 干货(xx 类) → ...

---

1/
(第一条)

2/
(第二条)

...

n/
(最后一条)
```

每条之间空行分开。条数 **5-9 条最合适**,少于 5 条读起来不像 thread,多于 10 条读者掉队。

## 常见错误

如果出现下面这些,重写:

- **第一条在介绍背景**("最近在做一个项目,想跟大家分享...")——不是钩子,是废话。把高潮前置。
- **第一条泛泛而谈**("一些关于 XX 的想法""聊聊 XX")——没价值主张,读者没理由点开第二条。
- **营销词堆砌**("最强""颠覆""重新定义")——换成具体数据和场景,让亮点自己浮现。
- **中间有过渡条**(只是"那么问题来了""接下来讲讲")——纯过渡条直接删,每条必须独立有价值。
- **每条都在讲道理,没例子**——至少要有一条是案例/数据,否则纯观点 thread 读起来很飘。
- **结尾在总结**("以上就是我对 XX 的看法")——改成互动邀请或预告。
- **"让我们"开头**——没人这么说话,删。
- **一条里三个以上的点**——拆。
- **同一种积木连续 3 条**——插一条其他类型调节奏。
- **emoji 抢戏**(一条里三四个,或每条都堆满)——减到 1-2 个。
